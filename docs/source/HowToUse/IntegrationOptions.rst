Возможности для интеграции
==========================

На следующей схеме показаны способы интеграции с Диадоком:

|image0|

Выбор способа работы с API Диадок зависит от решаемых задач, требований к клиентской части и доступных средств разработки.

Базовый уровень интеграции - **HTTP API**.
На этом уровне строятся платформо-независимые интеграционные решения. Это значит, что с HTTP API могут работать клиентские приложения, написанные на разных языках программирования и запускаемые под разными операционными системами.

Для клиентов, использующих технологии Microsoft, в SDK Диадока существует **.NET/COM-библиотека** :doc:`DiadocApi<../DiadocApiDll>`. Библиотека DiadocApi - надстройка над HTTP API. Она преобразует обращения к объектам и методам в HTTP-команды API Диадока. Кроме того, библиотека заключает в себе работу с криптографией, так что разработчикам клиентского приложения не придется разбираться в CryptoAPI.

Для программных продуктов на платформе 1С существует **1С модуль** (внешний компонент). Он решает задачи стыковки 1С-решений с Диадоком.
Внешний компонент написан на языке C++ и не содержит лишних зависимостей, поэтому работает на любой Windows-системе без установки дополнительных модулей.

Кроме этого, **SDK Диадока** доступен для языков Java и C++. Из языков Visual Basic и JavaScript можно работать с Диадоком через COM-сервисы.
Примеры работы с ними есть в составе SDK. SDK доступен для скачивания по `ссылке <https://diadoc.kontur.ru/sdk/>`__.
